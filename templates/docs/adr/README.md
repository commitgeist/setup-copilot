# Architecture Decision Records (ADR)

Este diretório contém os Architecture Decision Records do projeto.

## O que é um ADR?

Um ADR é um registro curto de uma decisão arquitetural significativa e
seu contexto. Serve para que futuros membros do time entendam **por que**
determinada decisão foi tomada.

## Convenções

- Numeramos sequencialmente: `ADR-001`, `ADR-002`, etc.
- Usamos o template em `TEMPLATE.md`.
- Um ADR pode substituir outro (campo `Status`).
